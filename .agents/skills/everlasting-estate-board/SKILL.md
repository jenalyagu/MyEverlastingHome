# Everlasting Estate Board Skill

Use this skill whenever generating, reviewing, or improving the My Everlasting Home estate concept board — including copy, image prompts, layout, and zone mapping.

## What This Skill Governs

1. **Board layout** — 7-section structure, spacing, section roles, visual rhythm
2. **Copy** — estate name, narrative, editorial note, footer tagline, section labels
3. **Image prompts** — hero exterior, floor plan watercolors, interior tiles, lifestyle tiles
4. **Zone mapping** — how form data fields map to floor plan rooms and lifestyle tiles

## Board Data Interface

```typescript
interface EstateBlueprintData {
  estateName: string           // AI-generated, e.g. "The Hawthorn Reserve"
  estateSubtitle: string       // 10–18 word subtitle
  estateNarrative: string      // 2-sentence editorial paragraph
  heroImagePrompt: string      // → Section 2 image
  heroImageUrl?: string
  editorialNote: string        // 8–14 word masthead note
  mainFloorZones: FloorZone[]
  upperFloorZones: FloorZone[]
  wellnessLevelZones: FloorZone[]
  sitePlanZones: FloorZone[]
  mainFloorImagePrompt: string      // → Section 3, panel 1
  upperFloorImagePrompt: string     // → Section 3, panel 2
  wellnessLevelImagePrompt: string  // → Section 3, panel 3
  sitePlanImagePrompt: string       // → Section 3, panel 4
  mainFloorImageUrl?: string
  upperFloorImageUrl?: string
  wellnessLevelImageUrl?: string
  sitePlanImageUrl?: string
  interiorLifestyleTiles: ImageTile[]   // 8 tiles → Section 4
  estateLifestyleTiles: ImageTile[]     // 10 tiles → Section 5
  scipBenefits: ScipBenefit[]           // always 6 → Section 6
  footerTagline: string         // 4–7 word motto → Section 7
  footerMonogram: string        // 2–3 uppercase letters
}
```

## Copy Generation

Copy fields (`estateName`, `estateSubtitle`, `estateNarrative`, `editorialNote`, `footerTagline`, `footerMonogram`) are generated by Claude via `POST /api/generate-copy`. The prompt sends full form data and expects raw JSON back.

Template fallbacks in `generateEstateBlueprint.ts` are used if the API call fails — they are intentionally generic and should not be shipped as final output.

## Image Generation

- All 23 images generated via `POST /api/generate-image`
- Server-side queue: `MAX_CONCURRENT=4`, 15s retry on rate limit
- Quality tiers: hero=`high`, all others=`medium`
- Provider default: `openai` (gpt-image-1)
- Cost per board: ~$1.55 at current quality tiers

## Common Issues & Fixes

**SCIP benefits section looks awkward (uneven row):**
→ Ensure `.eb-benefits-row` uses `grid-template-columns: repeat(6, 1fr)` — never `auto-fill`

**Tile strips have orphaned images:**
→ Ensure `.eb-tiles-row` uses `display: flex; flex-wrap: nowrap` — never grid auto-fill

**Image generation hitting rate limit:**
→ Server queue handles this automatically with 15s retry. If persistent, check OpenAI billing at platform.openai.com/settings/billing

**Copy is generic (template fallback showing):**
→ Check `ANTHROPIC_API_KEY` is set in `.env`. Restart the server after `.env` changes.

**Floor plans look like 3D renders instead of watercolor drawings:**
→ Remove any "render", "3D", or "CGI" language from floor plan prompts. Add "no photography, illustration only".
